# Как использовать .env файлы
Здесь коротко и мало об .env файлах

## Что такое .env файлы
.env файл — это файл куда можно сложить всю конфиденциальную информацию (логины, пароли, токены, ключи доступа и прочее) и дальше с этим работать. Код берёт эти данные из файла и больше никакой мароки. Нет надобности лезть в код и искать токен от своего API если тот внезапно утёк. Обычно .env добавляется в gitignore, чтоб не светить инфу.

## Как создать
Ответ удивит, но очень просто. Все данные хранятся в формате ключ-значение. Тоесть под капотом всё выглядит примерно так:

```env
API_TOKEN='123'
PASSWORD='456'
```
Дальше о том как использовать .env файлы в python-проектах
### Установите библиотеку python-dotenv
Эта библиотека позволяет загружать переменные окружения из файла .env.

`pip install python-dotenv`

### Создайте файл .env
Создайте файл с именем .env в корневой папке вашего проекта. В этом файле запишите свой секретный ключ:

`API_TOKEN="TOKEN"`

### Используйте переменные из .env в вашем проекте
В вашем Python-коде загрузите переменные окружения с помощью библиотеки dotenv.

```python
from dotenv import load_dotenv
import os

#Загрузить переменные из файла .env
load_dotenv()

#Получить значение переменной
api_token = os.getenv("API_TOKEN")

print(f"Your API Token is: {api_token}")
```

### Что дальше

Токен мы получили и дальше можем использовать его как хотим.
На последок скажу, что если файла .env нет в папке или название запрашиваемонекорректно, то функция getenv вернёт None. Это удобно использовать в проверках

```python
from dotenv import load_dotenv
from sys import exit as sexit
import os

# Загрузить переменные из файла .env
load_dotenv()

# Получить значение переменной
API_TOKEN = os.getenv("API_TOKEN")
if API_TOKEN is None:
    print('Отсутствует токен')
    sexit(1)  # Завершаем программу с кодом ошибки 1

...
```
